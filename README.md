# Smart-Agri-Agent 🌾

An aesthetically premium, modern, and highly interactive **Agriculture Handbook & AI Support Advisor** portal. This codebase features a unified static-friendly **React + Vite** frontend and a lightweight **Node.js + Express** backend, built to embed the official **IBM watsonx Orchestrate** chat channel dynamically.

---

## 🎨 Core Portal Features

1. **Dynamic Agricultural Dashboard**:
   - **Crop Matchmaker**: Interactive calculator filtering compatible crops based on your specific soil type, climate zone, and water availability.
   - **Meteorological Advisory Widget**: Displays specific real-time field guidelines (weeding, irrigation schedule offsets, pest alerts) based on sunny, rainy, cloudy, or windy weather selectors.
   - **Commodities Index**: Simulated tracker showcasing quintal prices for key regional crops (Wheat, Paddy, Maize, Potatoes).
2. **Exhaustive Agriculture Handbook**:
   - Curated, step-by-step guides for planting grains, vegetables, and preparing soil nutrients.
   - Accordion drawers covering sowing spacing, watering stages, fertilization, and integrated organic pest controls.
3. **Subsidies & Schemes Directory**:
   - Complete guides for major Indian policies (PM-KISAN, crop insurance PMFBY, and Paramparagat organic PKVY clusters).
   - **Policy Benefit Estimator**: Interactive calculator estimating crop insurance premium rates and organic inputs subsidies based on farm acreage.
4. **Helpline & Assistance Desk**:
   - Direct dialers to Kisan Call Center (KCC) and Pest Units, curated textbook libraries, and mock query tracking forms.

## 🤖 IBM watsonx Orchestrate Embed Integration

The application dynamically executes the raw `<script>` tags generated by your watsonx Orchestrate integration console.

* **Safe Mount Redirection**: Out of the box, the Orchestrate script uses `rootElementID: "root"`, which replaces the entire body of a React app. Our dynamic loader intercepts this and routes the rendering to a separate `#ibm-orchestrate-chat-mount` node. This keeps your React dashboard fully visible and lets the chat widget overlay float naturally in the bottom right corner.
* **React HMR Protection**: Built-in state checkers prevent multiple script injections during hot reloads or component updates.
* **Dynamic Fallback**: If no credentials are set, the portal defaults to an interactive offline demo chat widget.

## 🚀 How to Run Locally (Fullstack)

To run the application locally on your machine, you need to spin up the backend and frontend dev servers.

### 1. Configure Credentials
Open the [backend/.env](backend/.env) file and enter your configurations or paste the generated script tag at the bottom of the file:
```env
PORT=5000
IBM_ORCHESTRATE_INTEGRATION_ID=YOUR_INTEGRATION_ID
IBM_ORCHESTRATE_CRN=YOUR_CRN
IBM_ORCHESTRATE_AGENT_ID=YOUR_AGENT_ID
IBM_ORCHESTRATE_HOST_URL=YOUR_HOST_URL
IBM_ORCHESTRATE_AGENT_ENV_ID=YOUR_ENV_ID
```

### 2. Start the Backend
```bash
cd backend
npm install
npm start
```
*(The backend serves configs on [http://localhost:5000](http://localhost:5000))*

### 3. Start the Frontend
```bash
cd frontend
npm install
npm run dev
```
*(The frontend runs on [http://localhost:5173](http://localhost:5173))*

## ⚡ Standalone Static Deployment (Vercel / GitHub Pages)

This portal can be built and hosted purely as a static frontend with **no backend server running**. 

1. Create a [.env](frontend/.env) file inside the `frontend/` directory.
2. Define the configuration variables using the `VITE_` prefix:
   ```env
   VITE_IBM_ORCHESTRATE_INTEGRATION_ID=YOUR_INTEGRATION_ID
   VITE_IBM_ORCHESTRATE_CRN=YOUR_CRN
   VITE_IBM_ORCHESTRATE_AGENT_ID=YOUR_AGENT_ID
   VITE_IBM_ORCHESTRATE_HOST_URL=YOUR_HOST_URL
   VITE_IBM_ORCHESTRATE_AGENT_ENV_ID=YOUR_ENV_ID
   ```
3. Run the compiler:
   ```bash
   cd frontend
   npm run build
   ```
4. Push or upload the generated `frontend/dist` directory to your static hosting provider of choice (Vercel, Netlify, or GitHub Pages). Set the publish directory to `dist`.

## 🔒 Security & Widget Visibility

If the script loads but the widget is not visible on your website, you are likely hitting a **NoJwtError** authorization block. This means **Secure Web Chat** is turned on in your IBM console.
- **For Testing**: Go to your watsonx Orchestrate Console > Integrations > Embedded Agent settings, select **Security**, and toggle **"Secure your web chat"** to **OFF**. Refresh the page.
- **For Production**: Listen to the `authTokenNeeded` event in `wxOConfiguration` and supply a JWT signed with your private key from a server endpoint.
